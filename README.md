# BeatSync Video Editor

Автоматический создатель видеоклипов, синхронизирующих фото и видео с битами музыкального трека.

## Возможности

- 🎵 **Анализ музыки**: Автоматическое обнаружение битов и определение BPM с помощью librosa
- 📸 **Обработка фото**: Эффект Кен Бёрнс (масштабирование/панорамирование), вращение, фильтры
- 🎬 **Обработка видео**: Обрезка, зацикливание, цветовые фильтры (сепия, холодный/теплый оттенки)
- 🎯 **Синхронизация**: Переходы между клипами точно в момент ударов бита
- 🎨 **Эффекты**: Случайные переходы (fade, cut), цветовые фильтры
- 📤 **Экспорт**: Full HD 1920x1080, 30 fps, кодек H.264

## Требования

- Python 3.7+
- tkinter (обычно входит в состав Python)
- moviepy
- librosa
- numpy
- Pillow

## Установка

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/egoffn1/Autogenerator-of-edits.git
cd Autogenerator-of-edits
```

### 2. Установите зависимости

**Автоматическая установка** (рекомендуется):
Просто запустите программу - она сама проверит и установит все необходимые пакеты:

```bash
python beatsync_editor.py
```

**Ручная установка**:
```bash
pip install -r requirements.txt
```

Или по отдельности:
```bash
pip install moviepy librosa numpy Pillow scipy audioread
```

### 3. Системные требования

Для работы программы также необходимы системные пакеты:

**Ubuntu/Debian:**
```bash
sudo apt-get install python3-tk ffmpeg
```

**Fedora:**
```bash
sudo dnf install python3-tkinter ffmpeg
```

**macOS:**
```bash
brew install python-tk ffmpeg
```

**Windows:**
- tkinter обычно входит в состав Python
- ffmpeg скачайте с https://ffmpeg.org/download.html и добавьте в PATH

## Использование

### Запуск программы

```bash
python beatsync_editor.py
```

### Работа с интерфейсом

1. **Выберите папку с фото/видео** - укажите путь к папке с исходными медиафайлами (JPG, PNG, MP4, MOV, AVI)
2. **Выберите музыкальный трек** - укажите путь к аудиофайлу (MP3, WAV)
3. **Выберите папку для сохранения** (опционально) - по умолчанию создаётся папка `output`
4. Нажмите **"Создать видео"**

Программа автоматически:
- Проанализирует музыку и найдёт биты
- Распределит медиафайлы по тактам (4 бита = 1 такт)
- Применит эффекты к каждому клипу
- Синхронизирует переходы с битами
- Создаст финальное видео с наложенной музыкой

## Алгоритм работы

1. **Анализ аудио**: 
   - Загрузка аудиофайла через librosa
   - Обнаружение битов (beat tracking)
   - Определение BPM и длительности такта

2. **Подготовка медиа**:
   - Сканирование папки с файлами
   - Распределение файлов по тактам
   - Если файлов меньше чем тактов - повторение по кругу
   - Если больше - равномерное распределение

3. **Обработка клипов**:
   - Фото → эффект Кен Бёрнс + случайное вращение
   - Видео → обрезка/зацикливание + цветовой фильтр
   - Длительность каждого клипа = 4 бита (один такт)

4. **Сборка видео**:
   - Объединение всех клипов
   - Наложение переходов (fade in/out)
   - Добавление аудиодорожки
   - Рендеринг в H.264

## Примеры эффектов

### Для фотографий
- **Zoom In/Out**: Медленное приближение или отдаление
- **Pan Left/Right**: Панорамирование слева направо или наоборот
- **Rotate**: Лёгкое вращение (-2° до +2°)

### Для видео
- **Sepia**: Тёплый винтажный оттенок
- **Cool**: Холодный синеватый оттенок
- **Warm**: Тёплый желтоватый оттенок
- **B&W**: Чёрно-белый фильтр

## Настройки экспорта

- **Разрешение**: 1920x1080 (Full HD)
- **Частота кадров**: 30 fps
- **Видеокодек**: H.264 (libx264)
- **Битрейт видео**: 5000k
- **Аудиокодек**: AAC
- **Битрейт аудио**: 192k

## Обработка ошибок

Программа обрабатывает следующие ошибки:
- ❌ Папка с медиафайлами не найдена
- ❌ В папке нет поддерживаемых файлов
- ❌ Аудиофайл не найден или не читается
- ❌ Недостаточно памяти для обработки
- ❌ Ошибка при рендеринге видео

## Производительность

Время обработки зависит от:
- Количества медиафайлов
- Длительности аудио
- Разрешения исходных файлов
- Мощности компьютера

**Рекомендации**:
- Используйте файлы схожего разрешения
- Для длинных треков (>5 мин) убедитесь в наличии свободного места на диске
- Закройте другие ресурсоёмкие приложения во время рендеринга
